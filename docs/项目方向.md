# 项目方向

状态：已与用户确认，但 v1 范围已收紧。

## 项目一句话

面向 Claude Code 的轻量 Web/API 授权测试工作台：用少量高质量 skill、项目模板和 `pents` CLI，把一次渗透测试从 scope 到 finding、report、复盘反馈稳定记录下来。

`pents` 的定位不是单纯管理项目，也不是漏洞扫描器，而是 Claude Code 的渗透测试上下文压缩器、证据登记器和子代理协作辅助器。

## 已确认事实

- 以 Claude Code 为执行底座，不自研 agent loop。
- 不做 Xalgorix 式 Web UI / Dashboard；`refer/xalgorix/` 仅作为参考资料和 skill 原料。
- `skills/` 是正式武器库，不是原始 skill 的批量搬运目录。
- CLI 名为 `pents`，Python 实现，优先支持 pipx/uv tool 安装。
- v1 聚焦 Web 漏洞、API 安全、信息收集三类授权测试流程。
- 项目记录、漏洞记录、报告和复盘全部 Markdown 化。

## 目标用户

- 使用 Claude Code 做日常安全测试的渗透测试工程师。
- 需要稳定记录 scope、测试过程、finding、报告和复盘的安全研究员。
- 做靶场、Bug Bounty 或授权评估时，希望 AI 不乱跑、不漏记、不丢上下文的人。

## v1 产品目标

先做一个能真实跑完一次 Web/API 授权测试的最小工作台，而不是大而全的自动化平台。

v1 要解决的问题：

- 测试前：明确授权范围、目标、禁止动作和记录位置。
- 测试中：按阶段加载少量高质量 skill，边测边写 progress 和 finding。
- 测试后：从 finding 汇总 report，并把实战经验反馈到 skill 武器库。

## v1 必做范围

1. 模板优先
   - `templates/scope.md`
   - `templates/progress.md`
   - `templates/finding.md`
   - `templates/report.md`
   - `templates/review.md`

2. 核心 skill 治理
   - 不追求全量 300+ skill。
   - 不做批量格式转换；Claude Code 可以直接检索 `refer/xalgorix/` 原始资料。
   - 先整理 20-30 个 Web/API/Recon 核心 skill。
   - 每个 skill 必须包含：适用场景、授权边界、输入要求、执行步骤、停止条件、记录格式、验收方式。

3. 最小 `pents` CLI
   - `pents new`：创建项目目录和基础模板。
   - `pents finding`：按模板创建漏洞记录。
   - `pents report`：汇总 finding 生成报告草稿。
   - `pents review`：复盘项目记录，产出 skill 改进建议。

4. 顶层工作流规则
   - `CLAUDE.md` 约束 Claude Code 先读 scope、只在授权范围内行动、过程必须记录。

## CLI 设计哲学

参考 `D:\Backup\Documents\服务器远程运维\ai-board`：

- CLI 只做“稳定状态迁移”和“机械生成”，不替代 Claude Code 做智能判断。
- Markdown 给人和 AI 读；结构化状态尽量用简单、可验证的文件承载。
- 命令围绕生命周期设计，不堆扫描功能。
- 默认轻量，本地优先，不做云同步、权限系统、Web 登录。
- 不静默做高影响操作；涉及授权范围、外部网络、凭据、真实攻击流量时必须让用户确认。
- 每个命令都应该能被测试验证，并留下人能读懂的结果。

## pents 后续能力方向

`pents` 分三层演进：

- 管理层：`new`、`finding`、`report`、`review`，保证项目交付闭环。
- 提效层：`scope`、`brief`、`inventory`、`note`、`evidence`、`suggest-skills`，减少上下文重复读取和证据丢失。
- 协作层：`tasks`、`brief --agent <role>`、`merge`、`review-agent-output`，辅助 Claude Code 子代理并行工作。

子代理协作细节见 `docs/项目路线/pents与子代理协作路线.md`。

## v1 暂不做

- Web UI / Dashboard。
- 实时 WebSocket 事件推送。
- 自主调度、无人值守扫描、自动攻击链。
- 多用户、多租户、云同步。
- 数据库存储。
- 全量转换并维护 300+ skill。
- 把 `refer/xalgorix/` 的原始 skill 批量搬进 `skills/`。
- 把 `pents` 做成漏洞扫描器、代理工具或 exploit 执行器。
- 内置高风险攻击流量生成；需要时由授权测试者在 scope 内显式执行。

## v1 成功标准

用这套框架完成一次真实或靶场 Web/API 授权测试，产出：

- 完整的 `scope.md` 和 `progress.md`。
- 至少 3 个按模板记录的 finding。
- 一份由 `pents report` 汇总出的报告草稿。
- 一份 `review.md`，包含至少 1 条可落地的 skill 改进建议。
- 至少 20 个核心 skill 经过格式修正和质量治理。
