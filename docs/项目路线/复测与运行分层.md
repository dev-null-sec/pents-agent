# 复测与运行分层

## 目标

避免同一个项目多轮测试后文档越写越乱。

顶层文档记录“累计事实”和“最终交付”，每一轮测试记录到独立 `runs/` 目录。这样复测时可以看清：

- 本轮测了什么。
- 本轮新增了什么证据。
- 本轮哪些结论改变了。
- 本轮对总报告、finding、skill、CLI、字典有什么反馈。

## 推荐目录

```text
projects/<name>/
  scope.md
  inventory.md
  evidence.md
  findings/
  report.md
  review.md
  briefs/
  runs/
    R001-2026-06-03-active-recon/
      brief.md
      progress.md
      evidence.md
      outputs/
      raw/
      R001-report-delta.md
      review.md
```

## 顶层文档职责

| 文件 | 职责 | 更新时机 |
| --- | --- | --- |
| `scope.md` | 授权范围、窗口、速率、账号、禁止动作 | 授权变化时 |
| `inventory.md` | 累计资产、URL、API、参数、账号、测试面 | 每轮确认新资产后 |
| `evidence.md` | 累计证据索引，只保留可复用证据摘要和引用 | 每轮归并后 |
| `findings/` | 正式漏洞记录 | 主代理确认漏洞后 |
| `report.md` | 总报告或当前交付报告 | 每轮 delta 审核后 |
| `review.md` | 项目级复盘和路线反馈 | 每轮结束后归并摘要 |

## run 级文档职责

| 文件/目录 | 职责 |
| --- | --- |
| `brief.md` | 本轮任务目标、授权窗口、工具状态、执行主体 |
| `progress.md` | 本轮时间线、已执行、skipped/blocker、待确认 |
| `evidence.md` | 本轮证据、请求摘要、文件 hash、截图/输出路径 |
| `outputs/` | 工具输出、结构化结果、截图、报告草稿 |
| `raw/` | 原始响应、原始命令输出、下载文件；敏感信息需最小化 |
| `Rxxx-report-delta.md` | 本轮对总报告的新增/修改/撤销建议；文件名必须带复测编号 |
| `review.md` | 本轮对 skill、CLI、tools、dicts、模板的反馈 |

## 同步规则

每轮 Claude 执行结束后必须回答：

- 本轮是否需要更新顶层 `inventory.md`。
- 本轮是否需要把证据摘要合并到顶层 `evidence.md`。
- 本轮是否产生新 finding，或改变已有 finding 状态。
- 本轮 `Rxxx-report-delta.md` 哪些内容要合并进顶层 `report.md`。
- 本轮 `review.md` 哪些反馈要进入项目级 `review.md`、需求池、skill 或字典。

如果没有同步，必须写明原因：证据不足、等待用户确认、工具缺失、范围不明确、风险过高、重复发现、只做草稿。

## 复测编号

同一项目的报告增量和 run 目录必须带复测编号，避免多次复测后排序混乱。

编号规则：

- `R000`：初次基线或第一轮完整测试。
- `R001`：第一次复测或第一次专项补测。
- `R002`、`R003`：后续复测，按时间递增。

编号不得复用；如果同一天发生多轮，也继续递增编号，而不是只靠日期区分。

## 命名建议

`runs/<run-id>-<date-purpose>/` 使用编号、日期和目的：

- `R001-2026-06-03-active-recon`
- `R002-2026-06-03-setup-verification`
- `R003-2026-06-04-auth-api`
- `R004-2026-06-05-report-review`

每个 run 内的报告增量文件也使用同一个编号：

- `R001-report-delta.md`
- `R002-report-delta.md`
- `R003-report-delta.md`

如果生成正式复测报告快照，推荐放在 run 内：

- `R001-retest-report.md`
- `R002-retest-report.md`

## CLI 演进建议

后续可以加入：

- `pents run new <project> <name>`：创建 run 目录和模板。
- `pents run status <project>`：列出每轮状态。
- `pents merge <project> --run <name>`：把 run 结果合并到顶层文档。
- `pents report --delta <project> --run <name>`：生成本轮 report delta。

CLI 只做模板创建、状态读取和合并辅助，不直接执行扫描。
