# 主动扫描与字典闭环路线

## 目标

把主动 recon 纳入授权测试流程，同时保持可解释、可停止、可复盘。

这次路线调整有两个关键点：

- 子域名 DNS 枚举不按 HTTP 低频处理，否则会非常慢；它采用受控批量解析，核心是确认并发/速率、字典规模、resolver 和停止条件。
- 子域名字典先照搬 fuzzDicts 主字典，不急着自创精选小字典；优化、裁剪、分层是后续复盘任务。

## 字典取舍

- `refer/fuzzDicts/` 是原始字典参考库。
- `dicts/curated/subdomains-main.txt` 原样同步自 `refer/fuzzDicts/subdomainDicts/main.txt`，作为当前默认子域名字典。
- `dicts/curated/web-paths-small.txt` 和 `dicts/curated/params-small.txt` 仍用于低频 Web/API 发现和辅助分析。
- `dicts/candidates/` 是复盘回灌缓冲区，实战中发现的新词条先进入 candidates。
- 弱口令和 payload 类字典不进入默认主动 recon。

## 主动扫描分级

| 等级 | 场景 | 字典 | 节奏 | 是否默认 |
| --- | --- | --- | --- | --- |
| R0 | 被动收集 / 本地 JS 分析 | 无主动请求 | 无 | 默认 |
| D1 | 主动 DNS 子域名枚举 | `dicts/curated/subdomains-main.txt` | 受控批量，按用户确认的 DNS 并发/速率 | 推荐 |
| D2 | 扩展 DNS 枚举 | 从 `refer/fuzzDicts/` 选取更大或专项词表 | 需单独确认规模、resolver、并发/速率 | 非默认 |
| H1 | HTTP 存活和服务指纹 | 无路径字典 | 低频 | 推荐 |
| W1 | 目录/API 路径发现 | `dicts/curated/web-paths-small.txt` | 低频，小字典 | 条件满足后可用 |
| P1 | 参数名辅助分析 | `dicts/curated/params-small.txt` | 默认只读，不自动爆破 | 可用 |
| X1 | 弱口令 / exploit / payload | 禁止默认使用 | 需专项授权 | 禁止默认 |

## 主动子域名枚举触发条件

执行主动 DNS 子域名枚举前必须同时满足：

- scope 明确包含目标根域名和子域名。
- 用户确认授权窗口。
- 用户确认 DNS 并发或速率。
- 明确字典文件和词条规模，默认是 `dicts/curated/subdomains-main.txt`。
- 明确 resolver 来源：可信公共 DNS、用户提供 resolver，或其他已说明来源。
- 明确停止条件：DNS 限速、SERVFAIL/超时异常、泛解析噪声、命中第三方资产、窗口结束。

不要求实际 URL 才能做 DNS 子域名枚举；实际 URL 是 HTTP 存活、路径/API 探测、小范围端口确认和服务指纹的前置条件。

如果任何条件缺失，Claude 必须记录为 blocker，并说明缺失项，不得只写“未执行”。

## 需要低频的场景

以下动作必须低频：

- HTTP/HTTPS 存活探测、标题和技术栈识别。
- 目录/API 路径发现。
- 登录态接口探测。
- 小范围端口确认。
- 源站 IP 直连验证。
- 任何可能触发业务状态变化、认证、写操作或告警的请求。

## 工具准备

建议工具：

- `subfinder`：被动子域名收集。
- `dnsx`：DNS 解析验证。
- `httpx`：HTTP 存活与标题/技术栈探测。
- `shuffledns`：主动字典解析，默认只在确认字典规模、resolver 和 DNS 并发/速率后使用。
- `subzy`：接管风险提示，需谨慎使用，不能替代人工确认。

工具缺失时，Claude 或 `pents doctor-recon` 必须输出：

- 缺失工具名。
- 影响：哪些步骤不能执行。
- 本机安装建议。
- 替代方案：例如仅输出手工命令或仅做被动来源。

## 结果回填

主动 recon 结果必须回填：

- `inventory.md`：子域名、DNS、CDN、端口、服务指纹。
- `evidence.md`：命令摘要、字典文件、时间、响应摘要。
- `progress.md`：已执行项、跳过项、阻塞项和原因。
- `review.md`：字典命中质量、漏掉的新词条、建议回灌。

## 字典回灌

复盘时执行：

1. 从 inventory/evidence/progress 中提取新子域名、路径、参数。
2. 与 `dicts/curated/` 对比。
3. 没有命中的有效词条进入 `dicts/candidates/`。
4. 后续根据命中率、误报率和执行成本决定追加、分层或裁剪。

详见 `dicts/notes/promotion-rules.md`。
