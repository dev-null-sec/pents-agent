# 项目路线

> 长期方向放这里。注意：路线不是当前开发指令，`docs/计划看板.md` 才是当前执行依据。

## 路线列表

- `pents与子代理协作路线.md`：`pents` CLI 的提效定位、Claude Code 子代理分工和协作边界。
- `模板与数据结构.md`：项目模板清单、字段用途和 `pents` 命令映射。
- `主动扫描与字典闭环.md`：主动 recon 分级、精选字典、fuzzDicts 参考库和复盘回灌规则。
- `Sub2API类开源应用指纹与验证路线.md`：从 JS、路由、默认配置识别 Sub2API 类开源 Web/API 应用，并规划授权验证路线。
- `复测与运行分层.md`：同一项目多轮测试的 `runs/` 分层、顶层文档与单轮文档职责、报告增量同步规则。
- `工具与脚本资产结构.md`：`tools/` 自写脚本分类、第三方工具边界、CLI 与 tools 的职责分离。
- `资产图谱与入口卡片.md`：资产图谱实体关系、可测试入口卡片、授权状态和项目文档回填规则。

## 使用规则

- 新路线可以新建独立 Markdown 文件。
- 路线变化需要同步 `docs/计划看板.md` 的优先级。
- 重要方向取舍需要记录到 `docs/决策记录.md`。
