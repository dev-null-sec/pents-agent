## 概述

前端 JS 静态分析发现，应用包含完整的安装向导 API 端点（`/api/v1/setup/*`），包括状态检查、数据库连接测试、Redis 连接测试和执行安装。如果生产环境未禁用这些端点，攻击者可能：

- 获取数据库和 Redis 配置信息
- 重新执行安装流程，覆盖现有管理员账号
- 通过安装流程创建新管理员账号

## 影响

若安装向导在生产环境未禁用，攻击者可能：
- 获取数据库和缓存配置信息（通过 test-db/test-redis 错误消息泄露）
- 重新安装系统并创建新的管理员账号
- 完全接管平台

## 修复建议

1. 生产环境部署后禁用所有 `/api/v1/setup/*` 端点
2. 通过环境变量或配置文件标记 `INSTALLED=true`，安装完成后阻止 setup 路由
3. 即使 setup 状态 API 返回 404/403，也不应在响应中泄露配置信息
4. 考虑在反向代理层面（Nginx/Caddy）拦截 `/setup` 和 `/api/v1/setup/` 请求
