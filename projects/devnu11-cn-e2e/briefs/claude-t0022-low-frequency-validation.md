## 目标

对已发现且可解析的 `devnu11.cn` 子域名做低频 HTTP/CDN、服务指纹和小范围 Web 端口确认，验证 F-0001 相关的极小范围公开端点是否真实存在。

当前目标：

- `ai.devnu11.cn`
- `blog.devnu11.cn`
- `lk.devnu11.cn`
- `st.devnu11.cn`

`online.devnu11.cn` 当前是 NOERROR/NODATA 候选，没有 A/AAAA/CNAME 时不要做 HTTP 请求。后续新增的可解析 `devnu11.cn` 子域名可按同一低频规则纳入。

## 授权与边界

- 用户已确认：所有已发现子域名可测试。
- 用户已确认：请求速率由执行主体自定。
- 速率策略要求：使用低并发、安全区间和随机等待，避免固定机械节拍和突发压力。
- 随机等待用途：降低固定节拍和突发压力，不用于规避检测、绕过风控或扩大请求量。

不在范围内：

- 非 `devnu11.cn` 子域名。
- 源站 IP 直连验证，除非用户单独确认 IP 授权。
- DoS、压测、全端口扫描、大流量扫描。
- 目录爆破、大字典 fuzz、漏洞 payload 扫描。
- 破坏性写入、删除数据、持久化、社工、钓鱼、凭据填充。
- 未提供账号前的认证后 API、管理员权限、支付、OAuth 业务流验证。

## 建议速率

HTTP/CDN/服务指纹：

- 全局并发：1-2。
- 每个请求之间随机等待：2-8 秒。
- 每个子域名首轮请求预算：不超过 8 次。
- 采集内容：状态码、标题、最终 URL、跳转链、关键响应头、证书摘要、基础技术栈。

小范围 Web 端口确认：

- 端口范围：80、443、8080、8443。
- 全局并发：1。
- 目标之间随机等待：3-10 秒。
- 不做全端口扫描，不做服务爆破。

路径/API 极小范围验证：

- 只在已确认存活入口上验证。
- 候选路径：`/setup`、`/api/v1/setup/status`。
- 请求之间随机等待：5-15 秒。
- 不做目录爆破，不做参数 fuzz，不做写入请求。

遇到以下任一情况立即停止对应目标或整轮任务，并记录 blocker：

- 429、频繁 403、验证码升级、WAF 明显拦截。
