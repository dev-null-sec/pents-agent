# Claude Code 分场景 Recon 补强任务卡

## 任务

- 任务来源：ai-board `T-0020` / 后续 recon 补强
- 项目目录：`projects/devnu11-cn-e2e/`
- 授权目标：`*.devnu11.cn`
- 执行主体：Claude Code
- Codex 角色：补充流程、清单和交接材料，不代跑目标探测

## 背景

第一轮端到端验证已经完成 JS 静态分析和被动 recon，但信息收集基本盘仍不完整：

- 子域名发现不足，未做主动 DNS 子域名字典枚举。
- 未做已确认入口的端口探测。
- 未判断 CDN/WAF/代理层。
- 未沉淀源站线索挖掘流程。
- 未记录服务指纹、响应头、证书和跳转链。

本任务的目标是按授权边界补齐这些信息，并把结果写入 `inventory.md`、`evidence.md`、`progress.md` 和 `review.md`。

## 频率策略

不要把所有主动 recon 都叫“低频”。不同动作的风险和效率不同：

| 动作 | 是否可主动 | 推荐节奏 |
| --- | --- | --- |
| 被动来源查询 / 本地 JS 分析 | 可以 | 无主动请求 |
| 主动 DNS 子域名枚举 | 条件满足后可以 | 受控批量解析，按用户确认的 DNS 并发/速率执行 |
| HTTP 存活探测 | 条件满足后可以 | 低频 |
| 目录/API 路径发现 | 仅对已确认入口 | 低频，小字典 |
| 小范围端口确认 | 仅对已确认入口或授权 IP | 低频，小端口集 |
| 源站 IP 直连验证 | 默认不做 | 需要用户单独确认 IP 授权 |

## 执行前置条件

### 主动 DNS 子域名枚举

执行前必须确认：

| 条目 | 要求 |
| --- | --- |
| 根域名 scope | `devnu11.cn` 及其子域名在授权范围内 |
| 授权窗口 | 用户明确允许执行的日期和时间段 |
| DNS 并发 / 速率 | 用户明确允许的 DNS 解析节奏 |
| 字典规模 | 默认 `dicts/curated/subdomains-main.txt`，原样同步自 fuzzDicts `subdomainDicts/main.txt` |
| 解析器来源 | 使用可信公共 DNS 或用户确认的 resolver 列表 |
| 停止条件 | 限速、SERVFAIL、泛解析噪声、第三方资产、窗口结束 |

只要上述任一条件缺失，Claude 必须把主动 DNS 枚举记录为 blocker，并说明缺失项。

### HTTP / 端口 / 路径 / API 探测

执行前必须确认：

| 条目 | 要求 |
| --- | --- |
| 实际访问 URL | 至少 1 个具体入口，如 `https://example.devnu11.cn` |
| 授权窗口 | 用户明确允许执行的日期和时间段 |
| 允许请求速率 | 用户明确允许的低频请求节奏 |
| 测试账号 | 仅认证后 Web/API 测试需要；基础 HTTP 指纹可先不依赖账号 |

如果实际 URL、授权窗口或速率未确认，只允许做被动来源查询、本地文件分析和已授权的主动 DNS 枚举，不得进行 HTTP/端口/路径/API 主动探测。

## 禁止动作

- 不做 DoS / DDoS / 压测 / 稳定性测试。
- 不做无边界大字典子域名爆破，不跑未确认并发的 DNS 解析。
- 不做全端口扫描，不使用 masscan、zmap 或类似高速扫描。
- 不使用漏洞扫描脚本、NSE vuln 类脚本或自动 exploit。
- 不测试非 `devnu11.cn` 及其子域名的第三方资产。
- 不主动探测 CDN、云厂商或 SaaS 供应商基础设施。
- 不直接访问疑似源站 IP，除非用户明确确认该 IP 也在授权范围内。

## 建议流程

### 1. 工具检查

执行前先检查工具：

```bash
pents doctor-recon
```

如果缺少 `subfinder`、`dnsx`、`httpx`、`shuffledns` 或 `subzy`，输出必须包含：

- 缺失工具名。
- 影响：哪些 recon 步骤不能执行。
- 本机安装建议，区分 Windows / WSL / Linux / macOS。
- 替代方案：例如只做被动来源，或先生成待执行命令交给用户确认。

### 2. 子域名发现

先做被动来源，至少覆盖 2 类来源：

- 证书透明日志：crt.sh / Censys 证书搜索等。
- 被动 DNS / 威胁情报：OTX、urlscan、SecurityTrails、DNSDumpster、Shodan/Censys 可公开查询部分。
- 已有材料：JS 文件、HTML、配置片段、报错信息、证书 SAN、历史 URL。

满足主动 DNS 条件后，使用项目默认子域名字典：

- 默认字典：`dicts/curated/subdomains-main.txt`
- 来源：原样同步自 `refer/fuzzDicts/subdomainDicts/main.txt`
- 后续优化：根据实战命中率和误报率再拆分 fast / normal / deep，不在当前阶段自行发明小字典
- 结果回灌：发现有效但精选字典缺失的词条，写入 `dicts/candidates/subdomains.txt`

记录要求：

- 每个子域名的来源、采集时间、是否解析成功。
- 主动 DNS 使用的字典、词条数量、DNS 并发/速率、resolver 来源。
- 如果未做主动 DNS，按“跳过 / 阻塞反馈格式”写清原因。

### 3. DNS 解析与 CDN 判断

仅对候选子域名做解析验证：

- 先做 2-3 个随机不存在子域名的泛解析检测。
- 记录 A / AAAA / CNAME / NS / MX 中与 Web/API 入口有关的信息。
- 根据 CNAME、IP 归属、响应头和证书判断是否存在 CDN/WAF/反代。
- 如解析到 Cloudflare、Akamai、Fastly、阿里云、腾讯云、华为云等第三方网络，只记录供应商判断，不对供应商资产继续测试。

记录要求：

- CDN 判断只能写“可能/较可能/未见明显 CDN”，不要过度断言。
- 源站 IP 只能作为线索记录，未获用户确认前不得直接探测。
- 泛解析噪声必须单独标记，不得当成真实资产。

### 4. 存活入口与服务指纹

对已确认 URL 或已解析子域名做低频 HTTP/HTTPS 探测：

- 记录状态码、标题、跳转链、最终 URL。
- 记录关键响应头：`Server`、`X-Powered-By`、`Via`、`CF-Ray`、`X-Cache`、安全响应头。
- 记录 TLS 证书签发者、SAN、有效期。
- 记录明显技术栈线索，如 Vue/Vite、Nginx、Caddy、Cloudflare、API 网关等。

记录要求：

- 只做 HEAD/GET 级别的轻量访问。
- 如果响应包含敏感数据，停止并记录证据摘要，不展开访问。

### 5. 小范围端口确认

只有在实际 URL、授权窗口和速率都确认后，才做小范围端口确认：

- 默认只确认 Web/API 常见入口：80、443、8080、8443。
- 如用户明确允许，可额外确认 3000、5000、8000、9000 等开发/管理面板常见端口。
- 22、3306、6379、9200 等运维或数据库端口只登记为“建议用户自查”，不默认主动探测。

记录要求：

- 每个主机只记录开放/关闭/过滤状态、协议和自然暴露的 banner。
- 不进行服务漏洞探测，不跑版本漏洞匹配，不爆破认证。

### 6. 源站线索挖掘

只做被动和低频线索整理：

- 历史 DNS / 历史证书 / 历史 URL 中出现的 IP。
- JS、HTML、报错、公开配置中出现的 API base URL 或内网/公网地址。
- 邮件、SPF、MX、对象存储域名等旁路线索。
- CDN 前后响应差异、证书差异、跳转差异。

记录要求：

- 源站线索默认写为“待用户确认”。
- 未确认 IP 授权前，不进行直连验证。

## 跳过 / 阻塞反馈格式

Claude 每轮输出必须区分 DNS 枚举和 HTTP/API/端口探测，不要笼统写“主动扫描未执行”。

原因类型必须从以下列表中选择，可多选：授权缺失、工具缺失、目标无输入、网络失败、风险过高、范围外、等待账号、其他。

```text
主动 DNS 子域名枚举：<已执行 / 未执行>
使用字典：<dicts/curated/subdomains-main.txt / 未使用>
DNS 并发/速率：<用户确认值 / 未确认>
结果：<新增数量 / 无新增 / 未执行>
未执行原因：<缺少授权窗口 / 缺少 DNS 速率 / 缺少 resolver / 字典规模未确认>
下一步需要用户确认：<具体问题>

HTTP/端口/路径/API 主动探测：<已执行 / 未执行>
未执行原因：<缺少实际 URL / 缺少 HTTP 速率 / 缺少账号 / 风险过高>
下一步需要用户确认：<具体问题>
```

## 输出要求

更新以下文件：

- `inventory.md`：补充子域名、DNS/CDN、端口、服务指纹、源站线索表。
- `evidence.md`：记录每个来源、采集时间、请求摘要、字典文件和证据引用。
- `progress.md`：记录已执行项、跳过项、阻塞项。
- `review.md`：复盘 recon skill、工具链和字典命中质量。

如发现真实风险，只能在证据充足时创建 finding；证据不足时写为 candidate finding 或线索。

## 停止条件

- 未确认所需授权条件却需要主动探测。
- DNS resolver 大量超时、SERVFAIL、限速或泛解析导致结果不可用。
- 出现服务异常、WAF/IDS 明显告警、请求被频繁拒绝。
- 发现疑似敏感信息或生产凭据。
- 需要扩大到第三方资产、源站 IP、全端口或高风险 payload。
